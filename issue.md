# Next.js + Bun Full-Stack Scaffold Spec

A complete scaffold specification for a production-ready Next.js app using Bun as the runtime and package manager.

---

## Stack Overview

| Layer                     | Technology              | Purpose                             |
| ------------------------- | ----------------------- | ----------------------------------- |
| Runtime & Package Manager | Bun                     | Fast installs, test runner, runtime |
| Framework                 | Next.js 15 (App Router) | Full-stack, RSC + Server Actions    |
| Language                  | TypeScript              | Type safety across the stack        |
| Styling                   | Tailwind CSS v4         | Utility-first CSS                   |
| UI Components             | shadcn/ui               | Radix UI + Tailwind components      |
| Animation                 | Framer Motion           | UI animations                       |
| URL State                 | nuqs                    | URL search params as React state    |
| Server Actions            | next-safe-action        | Typesafe server actions             |
| Data Fetching             | TanStack Query v5       | Client-side caching & fetching      |
| Database                  | PostgreSQL (local)      | Local Postgres instance             |
| ORM                       | Drizzle ORM             | Typesafe SQL queries                |
| Validation                | Zod                     | Runtime schema validation           |
| Auth                      | Better Auth             | Session-based authentication        |
| Testing                   | Bun test                | Built-in test runner                |
| Deployment                | Vercel                  | Zero-config Next.js hosting         |

## Project Initialization

```bash
# 1. Create the Next.js app with Bun
bunx create-next-app@latest my-app \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --src-dir \
  --import-alias "@/*"

cd my-app

# 2. Install all dependencies
bun add drizzle-orm postgres
bun add better-auth
bun add next-safe-action zod
bun add @tanstack/react-query @tanstack/react-query-devtools
bun add nuqs
bun add framer-motion

# 3. Install dev dependencies
bun add -d drizzle-kit
bun add -d @types/node

# 4. Initialize shadcn/ui
bunx shadcn@latest init
```

When prompted by `shadcn init`, choose:

- Style: **New York**
- Base color: **Zinc**
- CSS variables: **Yes**

### `src/lib/actions/index.ts`

```ts
import { createSafeActionClient } from "next-safe-action";
import { auth } from "@/lib/auth";
import { headers } from "next/headers";

// Public actions (no auth required)
export const action = createSafeActionClient();

// Protected actions (auth required)
export const authAction = createSafeActionClient().use(async ({ next }) => {
  const session = await auth.api.getSession({
    headers: await headers(),
  });

  if (!session) {
    throw new Error("Unauthorized");
  }

  return next({ ctx: { session } });
});
```

### `src/middleware.ts`

```ts
import { NextRequest, NextResponse } from "next/server";
import { getSessionCookie } from "better-auth/cookies";

const protectedRoutes = ["/dashboard"];
const authRoutes = ["/login", "/register"];

export async function middleware(request: NextRequest) {
  const session = getSessionCookie(request);
  const { pathname } = request.nextUrl;

  const isProtected = protectedRoutes.some((r) => pathname.startsWith(r));
  const isAuthRoute = authRoutes.some((r) => pathname.startsWith(r));

  if (isProtected && !session) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  if (isAuthRoute && session) {
    return NextResponse.redirect(new URL("/dashboard", request.url));
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

## Example: Server Action with next-safe-action + Zod

```ts
// src/lib/actions/post.ts
"use server";

import { z } from "zod";
import { authAction } from "@/lib/actions";
import { db } from "@/lib/db";
import { posts } from "@/lib/db/schema";

const createPostSchema = z.object({
  title: z.string().min(1).max(200),
  content: z.string().min(1),
});

export const createPost = authAction
  .schema(createPostSchema)
  .action(async ({ parsedInput, ctx }) => {
    const post = await db
      .insert(posts)
      .values({
        title: parsedInput.title,
        content: parsedInput.content,
        userId: ctx.session.user.id,
      })
      .returning();

    return post[0];
  });
```

## Notes for the Agent

- Always use `bun` instead of `npm` or `pnpm` for all install and run commands.
- Keep all database logic inside `src/lib/db/`. Never import `db` directly in components — go through server actions or route handlers.
- Use `authAction` (not `action`) for any server action that requires the user to be logged in.
- All Zod schemas live in `src/lib/validations/`. Import them in both the client (for form validation) and the server action (for parsing).
- Route groups `(auth)` and `(dashboard)` are for layout organization — they don't affect the URL.
- The `middleware.ts` protects routes at the edge. Update `protectedRoutes` and `authRoutes` arrays as the app grows.
